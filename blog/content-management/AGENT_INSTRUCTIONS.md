# AIエージェント向け記事作成指示書

## 最重要原則

**このブログは「売るためのブログ」ではなく「理解を深めるためのブログ」である。**

- アプリの宣伝ではなく、知見の共有が第一目的
- 露骨なマーケティング文は禁止
- 読者の理解を助けることを常に優先

## 記事作成フロー

### 1. ガイドラインの読み込み（必須）

記事作成前に**必ず**以下を読み込む：

1. `guidelines/BLOG_STYLE_GUIDE.md` - 執筆・文体ルール
2. `guidelines/APP_CONTEXT.md` - アプリ情報（必要に応じて）

### 2. テンプレートの選択

| 記事タイプ | テンプレート |
|-----------|-------------|
| 新機能・アップデート | `templates/post-update.md` |
| 使い方・チュートリアル | `templates/post-guide.md` |
| Tips・コツ・理論 | `templates/post-tips.md` |

### 3. 記事の作成

`drafts/[slug].md` に保存。

**ファイル名ルール:**
- 英数字とハイフンのみ
- 日付は含めない（frontmatterで管理）
- 内容を端的に表す名前

例: `pitch-analysis-basics.md`, `breathing-technique.md`

### 4. フロントマター

```yaml
---
title: '記事タイトル'
description: '記事の説明'
pubDate: YYYY-MM-DD
tags: ['タグ1', 'タグ2']
draft: true
---
```

## 執筆時の必須ルール

### 著者人格

- 一人称は「私」
- 立場: Vocalis Studio 開発者
- 専門家だが上から目線にならない
- 実体験・開発経験を根拠として語る

### 文体

- 丁寧語（です・ます調）
- 短文中心（1文40字以内目安）
- 技術用語は噛み砕いて説明

### 事実と推測の区別

```
事実: 「〜です」
推測: 「〜と考えられます」「〜かもしれません」
経験: 「私の開発経験では〜」
```

### 禁止事項

- 架空の統計データ・研究引用
- 「絶対に」「必ず」「〜に違いありません」
- 「！」の多用
- SNS口調
- 押し付けがましい表現

## 記事構成（基本）

```markdown
# タイトル

## はじめに
- 読者の悩み・疑問を提示
- 記事で分かることを明示
- 自分の立場を一言で

## 結論（先出し）
- 要点を簡潔に

## 本文
### 見出し1〜3
- 理論・背景・実践

## まとめ
- 要点を箇条書き
- 関連記事への誘導

---
この記事を書いた人：Vocalis Studio 開発者
```
