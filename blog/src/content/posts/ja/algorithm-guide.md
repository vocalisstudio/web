## アルゴリズムとは

Vocalis Studioでは、録音した声のピッチを検出するために複数のアルゴリズムを用意しています。

![アルゴリズム設定画面](/images/ja/06_settings_algorithm.png)

それぞれ特性が異なるため、自分の声や練習スタイルに合ったものを選ぶと、より正確な分析ができます。

## 利用可能なアルゴリズム

### FCPE（推奨）

AIベースの最新アルゴリズムです。

| 特徴 | 説明 |
|------|------|
| 精度 | 非常に高い |
| 安定性 | オクターブエラーが少ない |
| 処理速度 | やや遅め |

ニューラルネットワークを使用した高精度な検出。ほとんどの場合、これを選んでおけば問題ありません。

**向いている場面**:
- 一般的な練習
- 正確な分析が必要な場合
- 初心者〜上級者まで幅広く

### pYIN（バリエーション4種）

確率的アプローチのアルゴリズムです。

| バリエーション | 特徴 |
|--------------|------|
| pYIN | 標準的なバランス |
| pYIN-High | 検出率重視 |
| pYIN-Balanced | バランス型 |
| pYIN-Aggressive | 最大検出 |

**pYIN**: 標準的な設定。多くの声質に対応。

**pYIN-High**: より多くのピッチを検出。静かな声や息混じりの声に有効。

**pYIN-Balanced**: 検出率と安定性のバランスを取った設定。

**pYIN-Aggressive**: 検出率を最大化。ノイズも拾いやすくなる点に注意。

**向いている場面**:
- FCPEで検出できない場合の代替
- 特殊な声質への対応
- 処理速度を優先したい場合

### YIN

シンプルで高速なアルゴリズムです。

| 特徴 | 説明 |
|------|------|
| 精度 | 標準的 |
| 安定性 | オクターブエラーが起きやすい |
| 処理速度 | 高速 |

古典的なアルゴリズムで、処理が高速。ただし、オクターブ上や下を誤検出することがあります。

**向いている場面**:
- 高速な処理が必要な場合
- 安定した声で歌う上級者
- FCPEやpYINで問題がある場合の代替

## アルゴリズム選びの目安

| 状況 | おすすめ |
|------|---------|
| 迷ったら | FCPE |
| 検出漏れが多い | pYIN-High または pYIN-Aggressive |
| オクターブエラーが気になる | FCPE または pYIN |
| 処理速度を重視 | YIN |
| 息混じりの声 | pYIN-High |

## 設定方法

1. ホーム画面から「設定」をタップ
2. 「アルゴリズム」セクションを開く
3. 使用したいアルゴリズムを選択

設定は次回の録音分析から反映されます。過去の録音を別のアルゴリズムで再分析することも可能です。

## よくある質問

### どのアルゴリズムが一番正確？

一般的にはFCPEが最も正確です。AIベースで、オクターブエラーも少なく、多くの声質に対応しています。

### 結果が変になる場合は？

アルゴリズムを変えてみてください。声質によっては、pYINの方が合う場合もあります。

### 処理が遅いと感じたら？

YINは最も高速です。ただし、精度とのトレードオフがあります。

## まとめ

- 迷ったら **FCPE** を選択
- 検出漏れが多い場合は **pYIN-High**
- 処理速度重視なら **YIN**
- 声質や目的に応じて使い分けると精度が向上
