# Vocalis Studio SNS運用方針

## 運用方針

| 項目 | 内容 |
|------|------|
| 目的 | アプリの信頼性・継続開発を示す / 検索経由・App Store経由の訪問者に安心感を与える |
| 投稿トーン | 落ち着いた・事実ベース・簡潔（例：「録音機能を改善しました」程度） |
| 更新頻度 | 週2〜3投稿で十分（新機能・改善・Tips・進捗報告） |
| 想定読者 | ボイストレーナー、声楽・ナレーション練習者、技術に興味のあるユーザー |

---

## 投稿カテゴリ例（構成テンプレート）

| カテゴリ | 内容例 | 投稿頻度 |
|----------|--------|----------|
| 機能紹介 | 「録音一覧で再生・削除が可能になりました」など | 週1 |
| 開発進捗 | 「次回アップデートではピッチ解析の表示を改良予定です」 | 週1 |
| 使い方Tips | 「録音後、再生ボタンで音程を確認できます」 | 隔週 |
| リリース報告 | 「Vocalis Studio v1.2.3をリリースしました」 | 随時 |
| お知らせ | 「App Storeでの配信を開始しました」など | 不定期 |

## デザイン方針（Instagram）

- 背景：白または淡いグレー基調（Apple風）
- スクリーンショット中心（UIを主役に）
- アプリアイコンと同系統のフォント・配色
- 投稿内で余白を十分に取り、無駄な装飾は避ける
- 統一感を出すため、テンプレート化（例：Canvaで1デザイン保存）

---

## 初月運用サンプル（テンプレート）

| 週 | X投稿内容 | Instagram投稿内容 |
|----|----------|-------------------|
| 1 | リリース告知（v1.0正式公開） | アプリ紹介＋スクショ |
| 2 | Tips：「録音後の再生方法」 | UIの使い方解説 |
| 3 | 次回アップデート予定（ピッチ解析改善） | 比較画像（旧UI→新UI） |
| 4 | バグ修正・安定化報告 | 開発風景やApp Store紹介 |

---

## 最低限あると良い準備

- アプリアイコン画像（正方形・透過背景）
- UIスクリーンショット数枚（ライト・ダーク両モード）
- App Storeリンク（固定ツイート／プロフィールに設置）
- 投稿テンプレート（CanvaやFigmaで統一）
